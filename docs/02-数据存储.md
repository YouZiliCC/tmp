
# 系统数据处理与存储

## 1. 模块概述与设计目标

-   **关系型存储（MySQL）**：负责存储论文主表、基础元数据以及剥离出的“研究设计”高浓度大文本。通过 B+ 树索引确保标量查询的高性能。
-   **向量化存储（ChromaDB）**：负责存储全文滚动切块（Chunks）。利用现代向量编码模型将文本映射为高维数学向量，并构建 HNSW（分层导航小世界图）空间索引，支撑语义检索。
-   **两库逻辑纽带**：依靠第一模块生成的唯一主键 `paper_id` 与外键 `chunk_id` 实现异构数据库之间的跨库逻辑关联。
-   **保存原则**：A 路保存“结构化元数据 + 大文本原文”，B 路保存“切分段原文 + 向量 + 定位信息”，保证后续检索、排序和引文回填都能直接取数。

## 2. 输入源

**来自模块一的增强型 JSON 数据包**

-   `paper_id`: 唯一论文标识符
-   `base_metadata`: 基础元数据（DOI、标题、出版年份等）
-   `research_design_text`: 研究设计章节大文本
-   `chunks`: 全文滚动切块文本块集合
-   `chunk_text`: 切分段原文，需要保留以便后续引文表直接回填
-   `chunk_id` / `offset` / `paragraph_index`: 切分段定位信息，用于引用回溯和展开展示

其中：

-   `base_metadata` 进入 MySQL 主表。
-   `research_design_text` 进入 MySQL 主表的大文本字段。
-   `chunks` 与 `chunk_text` 进入 ChromaDB。
-   `chunk_id`、`offset`、`paragraph_index` 作为 ChromaDB metadata 一并保存。

## 3. 数据流分流

输入数据包经系统分流为两路：

| 分流路径 | 处理内容 | 目标引擎 |
| :--- | :--- | :--- |
| **A路：标量与大文本分流** | `paper_id`、`base_metadata`、`research_design_text` | ORM Layer → MySQL |
| **B路：文本块纯文本流** | `paper_id`、文本块列表 | BGE-Large-Zh → ChromaDB |

```text
[A路：标量与大文本分流]
         │
         ▼
[关系型关系映射 (ORM Layer)]
         │
         ▼
[MySQL 关系数据库引擎]
         │
         ▼
┌─────────────────────────────────────────────────────────────┐
│  物理表: papers_master                                       │
├─────────────────────────────────────────────────────────────┤
│  * paper_id (PK)             │  主键标识                      │

│  * doi                       │  DOI标识符                     │
│  * publish_year              │  出版年份                      │

│  * author                    │  作者                          │
│  * keywords                  │  关键词                        │
│  * abstract                  │  摘要                          │

│  * ... (其他元数据字段)        │                              │
└─────────────────────────────────────────────────────────────┘
```

### A路保存内容说明

-   **保存位置**：MySQL 数据库中的 `papers_master` 表。
-   **保存内容**：论文主键、标题、DOI、年份、作者、关键词、摘要、来源期刊，以及完整的 `research_design_text`。
-   **用途**：支持结构化筛选、字段加权检索、论文详情展示和第四部分中的整段上下文注入。

### A路建议索引

-   `paper_id` 主键索引
-   `publish_year` 普通索引
-   `author` 普通索引
-   `doi` 唯一索引或普通索引
-   `title` / `keywords` / `abstract` / `research_design_text` 的全文索引

## 5. B路：向量化存储设计（ChromaDB）

```text
[B路：文本块纯文本流]
         │
         ▼
[BGE-Large-Zh 编码模型]
         │
         ▼
[1024维稠密向量]
         │
         ▼
[ChromaDB 向量数据库引擎]
         │
         ▼
┌─────────────────────────────────────────────────────────────┐
│  物理集合: paper_chunks_collection                           │
├─────────────────────────────────────────────────────────────┤
│  * paper_id                 │  关联论文主键                  │
│  * chunk_id                 │  文本块唯一标识                │
│  * chunk_text               │  切分段原文                    │
│  * document                 │  原始切片文本                  │
│  * vector                   │  1024维稠密向量               │
│  * offset                   │  原文起始偏移                  │
│  * paragraph_index          │  段落序号                      │
│  * ... (其他 metadata 字段)  │                              │
└─────────────────────────────────────────────────────────────┘
```

### B路保存内容说明

-   **保存位置**：ChromaDB 中的 `paper_chunks_collection` 向量集合。
-   **保存内容**：每个切分段的原文、向量、paper_id 绑定信息，以及 chunk 的位置元数据。
-   **用途**：支持第三模块的语义检索、第四模块的 Top-5 引文表回填，以及前端点击展开查看“具体段落”原文。

### B路建议索引与检索方式

-   `paper_id` 作为 metadata 过滤键。
-   `chunk_id` 作为文本块唯一键。
-   `vector` 进入 HNSW 索引。
-   查询时先按 `paper_id` 或白名单过滤，再做向量近邻搜索。
