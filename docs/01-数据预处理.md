## 1. 输入源 (Input Sources)

系统接收两种类型的输入数据：

-   **外部基础元数据 (JSON)**
    -   内容：DOI、标题、关键词等
    -   绑定方式：通过文件名/映射表
-   **1000篇原生Word文献 (.docx)**
    -   结构：标准学术论文结构排版
    -   输入方式：输入文件流

## 2. 处理核心 (Processing Core)

**核心引擎：python-docx 引擎**

-   **模块 1：全文读取与结构扫描 (Document Parser)**
    -   功能：解析 Word 文档，读取全文并识别基本结构。

经过核心处理后，数据流分为两个分支：

## 3. 流向分支 A：定向章节切片

-   **模块 2：学术章节状态机检测器 (Regex State Machine)**
    -   **触发开关**：检测“研究设计”等章节起始标记。
    -   **截止开关**：检测“模型分析”等章节结束标记。
-   **输出产物**：**粗粒度整段文本对象**
    -   示例：`research_design_text` (包含完整的研究设计章节内容)

## 4. 流向分支 B：全文滚动切块

-   **模块 3：滑动窗口文本切分器 (Sliding Window)**
    -   **窗口大小**：500 字
    -   **重叠大小**：100 字
-   **输出产物**：**二维映射文本块集合**
    -   结构：`[chunk_id, text, index]`

## 5. 统一数据封装层 (Data Packaging)

两个分支的输出在此汇合并整合：

-   **输出格式**：标准增强型论文数据对象 (Unified JSON Schema)

## 6. 最终输出 (Final Outputs)

封装后的数据分别输送至不同的存储模块：

-   **输出至 MySQL**：存入学术主表及设计大字段 (存储结构化元数据和完整章节文本)
-   **输出至 ChromaDB**：先由 BERT 模型将文本转化为向量，再存入向量库 (用于语义检索和相似度匹配)
