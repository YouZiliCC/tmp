# 部署指南

> 本文覆盖两种部署：**本地开发**（一键 dev）与**单机生产/远端服务器**（裸机或 Docker）。
> 数据（上千篇 docx + CSV，约 300MB）不进 git，需用 scp/tar 单独分发。

## 0. 前置依赖

| 工具 | 版本 | 用途 | 安装 |
| --- | --- | --- | --- |
| Go | ≥1.22 | 后端（modernc/sqlite 纯 Go，无需 cgo） | 官网或 `brew install go` |
| uv | 任意 | Python venv 与依赖管理 | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| fnm | 任意 | Node 版本管理（读 `frontend/.nvmrc`，默认 22） | `curl -fsSL https://fnm.vercel.app/install \| bash` |
| Node | ≥18（推荐 22） | 前端构建 | `fnm install 22` |

> 没有 uv/fnm 时脚本会自动回退到 `python3 -m venv`/`pip` 与 PATH 中的 `npm`。

LLM 与嵌入：
- LLM 走火山方舟（Volces Ark）DeepSeek，OpenAI 兼容协议，`.env` 配 `LLM_API_KEY/LLM_BASE_URL/LLM_MODEL`。
- 嵌入走**本地** `BAAI/bge-small-zh-v1.5`（火山方舟无兼容 `/embeddings`），首次 ingest 会自动下载模型（约 100MB）。

## 1. 本地开发（一键）

```bash
git clone <repo> xcj-dev && cd xcj-dev
cp .env.example .env            # 按需改 LLM_API_KEY（默认已填一组火山方舟 key）

# 首次摄取数据（data/ 已就位的前提下）
bash scripts/ingest.sh          # 全量 + 嵌入；想快速跑通可加 --no-embed 或 --limit 100

# 一键启动三件套（Python 8001 / Go 8080 / Vite 5173）
bash scripts/dev.sh
# 浏览器打开 http://localhost:5173
```

`scripts/dev.sh` 会用 uv 建 venv、fnm 切 Node、起 uvicorn + go run + vite，并 `tail -F` 三份日志；Ctrl-C 触发 trap 清理全部子进程。

## 2. 远端服务器部署（裸机，本仓库当前采用）

服务器：Ubuntu 24.04，已装 uv / fnm / Go 1.22 / Node 22。端口避开常用段，用 **19080(Go) / 19001(Py) / 19173(Vite)**。

### 2.1 拉代码

```bash
ssh <server>
git clone <repo> ~/tmp && cd ~/tmp     # 或 git pull 更新
```

### 2.2 写 .env（端口 + 凭据）

```bash
cat > .env <<'EOF'
API_ADDR=:19080
PY_SERVICE_URL=http://127.0.0.1:19001
LLM_API_KEY=<你的火山方舟 key>
LLM_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
LLM_MODEL=deepseek-v3-2-251201
LLM_TEMPERATURE=0.2
EMBED_BACKEND=local
EMBED_MODEL=BAAI/bge-small-zh-v1.5
DB_PATH=/root/tmp/data/storage/papers.db
DB_DSN=/root/tmp/data/storage/papers.db
EOF
```

### 2.3 分发数据（不走 git，用 tar over ssh）

在**本地**仓库根执行：

```bash
ssh <server> 'mkdir -p ~/tmp/data/word ~/tmp/data/storage; rm -rf ~/tmp/data/word/*'
# macOS 务必加 COPYFILE_DISABLE=1，否则会把每个 docx 的 AppleDouble 伴随文件（._xxx.docx）一并打包，
# 污染数据目录（python-docx 会对这些垃圾文件全部报 PackageNotFoundError）。
COPYFILE_DISABLE=1 tar -C data -cf - WanFangdata.csv word | ssh <server> 'cd ~/tmp/data && tar -xf -'
# 校验：ssh <server> 'ls ~/tmp/data/word/*.docx | wc -l'
# 若已误传，清理：ssh <server> 'find ~/tmp/data/word -name "._*" -delete'
```

> tar 流式传输比逐文件 scp 快很多（避免上千次握手）。ingest 侧也已显式跳过 `._`/`~$` 文件，双保险。

### 2.4 装依赖

```bash
ssh <server> 'cd ~/tmp
  uv venv .venv
  uv pip install --python .venv/bin/python -r pyservice/requirements.txt   # 含 sentence-transformers + torch(CPU)
  eval "$(fnm env)"; (cd frontend && fnm use --install-if-missing && npm install)
'
```

### 2.5 全量摄取（含嵌入）

```bash
ssh <server> 'cd ~/tmp
  nohup .venv/bin/python -m pyservice.ingest --data-dir ./data --db ./data/storage/papers.db > ingest.log 2>&1 &
'
# 1001 篇 CPU 嵌入约 15-40 分钟。tail -f ingest.log 看 progress N/1001。
```

> 仅本轮端口本地化需要的两处改动（仓库默认 8001/5173）：用 tmux 跑 dev.sh 前，把 `scripts/dev.sh` 里 uvicorn 端口改 19001、vite 端口改 19173，把 `frontend/vite.config.ts` 的 proxy target 改 19080。或直接用 2.6 的显式启动方式绕开 dev.sh。

### 2.6 启动服务（tmux 常驻）

```bash
ssh <server>
tmux new -s xcj            # 或 tmux attach
cd ~/tmp && bash scripts/dev.sh   # 端口已 patch 为 19xxx
# Ctrl-b d 脱离 tmux，服务继续运行
```

### 2.7 对外访问

服务器若无公网 IPv4，用 SSH 隧道：

```bash
ssh -L 19173:localhost:19173 -L 19080:localhost:19080 <server>
# 本地浏览器开 http://localhost:19173
```

绑定自定义域名（如 `xcj.youzl.io`）时，`vite.config.ts` 已设 `allowedHosts: true`，反代直接生效。

## 3. Docker 部署（可选，一键）

```bash
cp .env.example .env
docker compose build
docker compose up -d                       # pyservice:8001 / backend:8080 / frontend:8088
docker compose --profile ingest run --rm ingest   # 一次性容器跑摄取
# 浏览器开 http://localhost:8088
```

数据通过 volume 注入：`./data/word`、`./data/WanFangdata.csv` 只读挂载，`./data/storage` 读写共享，bge 模型缓存在 named volume `hf_cache`。详见 `docker-compose.yml`。

---

## 4. 数据更新后如何重摄

```bash
# 1) 本地更新 data/ 后，重新分发（见 2.3）
# 2) 远端重摄并热重载
ssh <server> 'cd ~/tmp
  rm -f data/storage/papers.db data/storage/papers.db-*
  .venv/bin/python -m pyservice.ingest --data-dir ./data --db ./data/storage/papers.db
  curl -fsS -X POST http://127.0.0.1:19080/api/reload    # 通知后端热重建索引（也可重启 dev.sh）
'
```

## 5. 排错

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| 智能检索「语义榜无结果」 | 未带嵌入摄取 / 未装 sentence-transformers | 装依赖后 `--no-embed` 去掉重新 ingest |
| stats 年份全 0、作者空 | CSV 编码被误判 | 已强制 GB18030/GBK 优先；确认 CSV 为万方导出 |
| 端口被占启动失败 | 旧进程残留 | `fuser -k 19080/tcp 19001/tcp 19173/tcp` 后重启 |
| LLM 接口 502 | key 失效 / 网络 | 检查 `.env` 的 `LLM_API_KEY`、服务器外网连通性 |
| 详情页全文为空 | 该 docx 解析失败或 raw_body 空 | 看 ingest.log 对应论文是否 fail |
| 首次 ingest 很慢 | 下载 bge 模型 + CPU 嵌入 | 正常，1001 篇约 15-40 分钟，一次性 |
